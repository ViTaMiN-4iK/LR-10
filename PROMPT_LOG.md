# План лабораторной работы №10: FastAPI vs Gin

## 1. Цель
Реализовать два микросервиса (Go/Gin и Python/FastAPI), обеспечить их взаимодействие через REST (JSON), реализовать graceful shutdown и контейнеризацию.

## 2. Этапы выполнения

### Этап 1: Базовый Go (Gin) сервис (Средняя сложность)
- **Структура проекта:**
  `go-service/`:
    - `main.go` (точка входа, graceful shutdown)
    - `handlers/` (обработчики)
    - `models/` (структуры данных)
    - `go.mod`
- **Эндпоинты:**
    1. `GET /health` – проверка работоспособности.
    2. `POST /items` – создание элемента (принимает JSON: `{"name": "string", "price": float}`).
    3. `GET /items/:id` – получение элемента по ID (in-memory хранение).
- **Graceful Shutdown:**
    - Использование `signal.Notify` для `SIGINT` и `SIGTERM`.
    - Использование `context` и `srv.Shutdown` для ожидания завершения активных соединений.
- **Тестирование:**
    - Написать тесты для хендлеров с использованием `httptest`.

### Этап 2: Базовый Python (FastAPI) сервис (Средняя сложность)
- **Структура проекта:**
  `python-service/`:
    - `main.py`
    - `routers/` или `handlers.py`
    - `schemas.py` (Pydantic модели)
    - `requirements.txt`
- **Эндпоинты:**
    1. `GET /health` – проверка работоспособности.
    2. `POST /items` – создание элемента (аналогично Go, но храним в отдельном словаре для чистоты сравнения).
    3. `GET /items/:id` – получение элемента.
- **Graceful Shutdown:**
    - Использование `lifespan` контекстного менеджера (FastAPI стандарт) для корректного закрытия при получении сигнала.
- **Тестирование:**
    - Написать pytest тесты с использованием `TestClient`.

### Этап 3: Взаимодействие сервисов (Средняя сложность)
- **Задача:** Python сервис выступает клиентом для Go сервиса.
    - Добавить в Python сервис эндпоинт `GET /proxy-items/{id}`.
    - При вызове этого эндпоинта, Python должен сделать HTTP запрос (через `httpx`) к Go сервису (`http://go-service:8080/items/{id}`) и вернуть результат.
- **Цель:** Продемонстрировать передачу сложных структур JSON между разными экосистемами (Go struct -> Python dict).

### Этап 4: gRPC (Повышенная сложность)
- **Реализация:**
    1. Создать `proto/items.proto` с сервисом `ItemService` (метод `GetItem`).
    2. Сгенерировать код для Go (сервер) и Python (клиент).
    3. Добавить в Go сервис gRPC-сервер на порту `50051` (параллельно с Gin).
    4. Добавить в Python сервис gRPC клиент и новый эндпоинт `GET /grpc-items/{id}`, который вызывает Go через gRPC.

### Этап 5: Docker Compose (Повышенная сложность)
- **Контейнеризация:**
    - Написать `Dockerfile` для Go (multistage build).
    - Написать `Dockerfile` для Python.
    - Создать `docker-compose.yml` с двумя сервисами (`go-app`, `python-app`) и общей сетью `backend`.
    - Убедиться, что `python-app` может резолвить `go-app` по имени сервиса.

### Этап 6: Итоговое тестирование и документирование
- Запуск всей инфраструктуры через `docker-compose up`.
- Проверка REST взаимодействия.
- Проверка gRPC взаимодействия.
- Проверка graceful shutdown при `docker-compose down`.
- Формирование `PROMPT_LOG.md`.
